Security Controls and Planning

Security control refers to the set of protocols, policies, practices, etc. that is taken in order to fulfill cybersecurity. They are such controls taken to protect digital devices from any vulnerabilities and threats. They are necessary for the following reasons:

  • They help to protect digital assets from external threats.
  • They help protect sensitive data from potential leak or breach.
  • They also fulfill regulatory compliance.
  • They ensure CIA of information.
  • They minimize risks and maintain workflow in an organization.

Types of Cybersecurity Controls

  1. Physical controls: It involves security controls developed to protect physical infrastructure and assets from damage, theft or unauthorized access. Example, CCTV surveillance, Biometrics entry system, Locks in gates and windows, Security Guards.
  2. Procedural controls: It involves policy, practice and processes to be followed by human to ensure cybersecurity. Example, security awareness, data handling plans, Incident Response plans.
  3. Legal controls: It involves the laws, rules and regulations, compliance that an organization should follow for cybersecurity. Example, HIPPA compliance, contracts and agreements, GDPR.
  4. Technical controls: It involves security measures implemented through technology. Example, firewalls, Access Control Systems, IDS, IPS, Anti malwares, Encryption.

Cybersecurity Defense Approach

The four step approaches for cybersecurity defense are:

  1. Identify: It is the first step which involves identifying the assets in an organization which are vulnerable to threats. The assets include data and information, system, network, application, etc. It also involves listing other resources that can be indirectly harmed.
  2. Assess: It is the next step which involves analyzing. The identified risks and threats are analyzed to understand how it can affect the system and to what extent. Also, the risks need to be prioritized to figure out what should be deal with first.
  3. Protect: It includes the protective measures. Implementing policies, firewalls, IDS and IPS, biometrics and multi-factor authentication, etc. fall in this step. The identified risks should be mitigated by adopting effective security measures.
  4. Monitor: This step involves continuous monitoring of system and network to detect any suspicious or malicious action. It involves monitoring logs and also user activities to detect any social engineering.

Business Continuity Plan (BCP)

BCP (Business Continuity Plan) is a set of plans and policies that mention how an organization will function in case of incidents such as disasters, cyberattacks, etc. which can disrupt the organization assets. BCP addresses the following things:

  • Identify risks (threats and vulnerabilities) and create identify-assessment prioritize plan.
  • Analyze the possible losses of assets and how they can impact the organization.
  • Develop response, recovery and post-incident plans.
  • Develop communication plans.
  • Develop backups for important data.

Disaster Recovery Plan (DRP)

DRP (Disaster Recovery Plan) is a set of plans that outlines how an organization respond to and recover from any unfortunate events or disasters. This plan ensures minimal downtime, protects critical assets, and maintains business continuity. Actually, DRP is a subset of BCP.

The Disaster Recovery Planning addresses following things:

  • Identify threats and assess how downtime in IT resources would affect the organization operations.
  • Assess recovery objectives: Recovery Time Objective (RTO), i.e. how fast the system must be restored and Recovery Point Objective (RPO), i.e. how much data loss is acceptable.
  • Identify backup strategies.
  • Assign roles and responsibilities. For example, an employee can be assigned to restore data from a backup site.
  • Develop recovery procedures.

Incident Response Plan

Incident Response Plan is a set of plans to respond to a cyberattack. The goal of incident response planning is to minimize the impact, reduce recovery time and ultimately reduce losses.

There are six phases of incident response planning:

  1. Preparation: It involves setting up tools, technology, policies, etc. Employees should be trained to properly respond to cyberattacks. An incident response team should be built.
  2. Identification: Monitoring tools should be set up to identify any incidents. Logs should be monitored and when any suspicious activity is detected, an alert should be made.
  3. Containment: When an incident is detected, the compromised system should be isolated and the networks should be disconnected. This is done to limit the area of impact.
  4. Eradication: The malware should be removed and the compromised files should be cleaned up. Patches should be applied to fix the vulnerabilities in the system or application.
  5. Recovery: The affected systems should be restored. The credentials should be reset. The system should be tested to ensure it is secure and working properly.
  6. Research: The incident should be researched and lessons should be learned.
